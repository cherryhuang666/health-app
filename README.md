# 我的健康档案 · Health App

将原本手动填写的 **就医头痛记录表.xlsx** 数字化成一个移动优先的网页应用。

## 功能模块

| 模块 | 说明 |
| --- | --- |
| 🏠 首页 / 仪表盘 | 今日记录、关键指标、近4周痛苦程度热力图、最近看诊、保险概览 |
| 📝 每日记录 | 用滑块快速记录头痛 / 腰背痛 / 尾骨 / 情绪等 0–10 度痛苦程度，勾选当日用药、写摘要 |
| 💉 疫苗 | 接种日期、剂次、效期（如"管5年"）、地点、备注 |
| 🏥 看诊 | 按 16 个科室分类（妇产 / 神内 / 复健 / 乳房 / 胸腔 / 甲状 / 中医 / 耳鼻 / 牙科 / 眼科 / 身心 / 外科 / 胃肠 / 骨科 / 皮肤 / 其他），记录医师、检查结果、用药、回诊 |
| 📑 保险理赔 | 南山 / 全球 / 健保局 等多家保险，金额、状态（待提交 / 已提交 / 已核准 / 已到账 / 拒赔）、累计统计 |
| 📊 分析 | 痛苦程度趋势折线图、各部位平均痛苦、用药频率、看诊分布饼图 |
| 🔍 搜索 | 跨所有模块全文搜索 |
| 📥 导入 / 📤 导出 | 一键从原 Excel 导入历史数据；可导出为 .xlsx 或 .json 备份 |

## 快速开始

### 1. 在电脑上使用

直接双击 `index.html` 用浏览器（推荐 Chrome / Edge）打开即可。

如果遇到导入 Excel 报错，建议用一个简单的本地服务器：

```powershell
# 任选其一
npx serve .
# 或
python -m http.server 8080
```
然后访问 `http://localhost:8080`。

### 2. 在手机上使用

把 `health-app` 文件夹放到任意静态托管服务上（GitHub Pages / Vercel / Netlify / Cloudflare Pages 都免费），用手机浏览器打开：

- **iPhone (Safari)**：底部 ⬆️ 分享按钮 → "添加到主屏幕"
- **Android (Chrome)**：右上角菜单 → "添加到主屏幕" 或 "安装应用"

之后就会有一个图标和原生 App 一样使用，离线也能记录（数据保存在手机浏览器本地）。

### 3. 从现有 Excel 导入历史数据

1. 打开应用 → 右上角 ⋯ → "从 Excel 导入"
2. 选择 `就医头痛记录表.xlsx`
3. 应用会自动识别"日期 / 头痛 / 腰背痛 / 摘要 / 疫苗 / 各科室 / 全球理赔 / 健保局"等列，转换为对应模块的记录

### 4. 让别人（或新手机）一打开网站就有你的历史数据（可选）

网页无法把 Excel 打进「安装包」，但可以把 **与应用「导出 JSON」相同格式** 的文件放在网站根目录：

1. 在本机：把 Excel 导入应用后，打开 **更多 → 导出 JSON**，得到一份备份文件。
2. 将文件**重命名为** `seed-data.json`，放到与 `index.html` **同一层**（仓库根目录）。
3. `git add seed-data.json`、`commit`、`push`。之后 **第一次**用某台设备打开网址、且该设备**从未存过本 App 数据**时，会自动加载并写入浏览器本地。

若没有 `seed-data.json` 或文件里没有日记/看诊等记录，应用行为与现在一样（空白起步）。

**隐私**：`seed-data.json` 会随静态网站一起公开下载（与把 Excel 放进公开仓库类似）。若介意，请用 **私密仓库 + 受支持的 Pages 方案**，或不要提交该文件、仍用手动导入。

## 数据安全

- 所有数据**只保存在你的本机浏览器** (`localStorage`)，**不会上传任何服务器**
- 建议定期通过 "导出 JSON 备份" 备份数据
- 清除浏览器数据 / 重装系统前请务必先导出

## 技术栈

- 纯 HTML + 原生 JavaScript（无构建工具）
- Tailwind CSS (CDN)
- Chart.js (图表)
- SheetJS / xlsx (Excel 导入导出)
- Service Worker (离线支持)
- Web App Manifest (可安装为 PWA)

## 文件结构

```
health-app/

├── app.js           # 全部应用逻辑

```

## 未来转原生 App

当前是 PWA（渐进式 Web 应用）。如要转成真正的原生 App，推荐路径：

1. **最快方式**：直接用 PWA 安装到主屏幕，体验已接近原生
2. **跨平台**：用 [Capacitor](https://capacitorjs.com/) 把当前网页打包成 iOS / Android 应用，几乎不用改代码
3. **重写为原生**：把数据模型迁移到 React Native / Flutter，本应用的所有 UI 布局与数据结构都可以复用
